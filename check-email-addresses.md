## 확인 방법

### 1. 면접관 정보 확인
웹 인터페이스에서 면접관 관리 페이지로 이동하여:
- 김희수 면접관의 이메일 주소 확인
- 정주연 면접관의 이메일 주소 확인
- 이메일 주소에 공백이나 특수문자가 있는지 확인

### 2. 테스트 이메일 발송
API를 사용하여 각 면접관에게 테스트 이메일 발송:

```bash
# 김희수 면접관 ID 확인 후
POST /api/interviewers/{면접관ID}/test-email

# 정주연 면접관 ID 확인 후
POST /api/interviewers/{면접관ID}/test-email
```

### 3. 서버 로그 확인
서버를 재시작하고 새 면접을 생성한 후 로그에서 확인:
- `📧 Recipient` 부분에서 각 수신자별 상세 정보
- `Email bytes` 부분에서 숨겨진 문자 확인
- `Accepted/Rejected` 상태 확인

### 4. 가능한 원인
1. **이메일 주소 오류**: Excel에 저장된 이메일 주소가 정확한지 확인
2. **스팸 필터**: 각 수신자의 스팸 필터 설정이 다를 수 있음
3. **메일 서버 정책**: 특정 계정에 대한 정책이 다를 수 있음
4. **이메일 계정 상태**: 계정이 비활성화되었거나 존재하지 않을 수 있음
